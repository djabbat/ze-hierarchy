## 3. Методология

### 3.1. Симуляция
- ✅ Написана на Python, 120 ботов
- ✅ HI = 0.532 (seed=0)
- ⚠️ Но: средний HI по 30 сидам = 0.497 (p = 0.49)
- **Проблема:** Эффект seed-зависимый, статистически не значим

### 3.2. Эксперимент
- RC-таймер: V₀·exp(-t/115 дней) — корректно
- LM393: порог 2.5 В — ±5% разброс → 120 шт × 5% = 6 шт за пределами спецификации
- ESP-NOW: 120 устройств × 10 мс слот = 1.2 с цикл — корректно
- Камера 4K: 120 точек — корректно

### 3.3. Статистика
| Параметр | Значение | Комментарий |
|---|---|---|
| N | 120 | 2 группы × 60 |
| Power | 0.80 | При d = 0.5 |
| α | 0.05 | Корректно |
| Тесты | A–F | 6 контрольных тестов |

**Проблема:** Power analysis предполагает d = 0.5, но симуляция даёт d ≈ 0.33. Нужно n = 120 на группу (240 ботов).

## 5. Вывод для фонда

**Вердикт:** Условное финансирование (Tier 2)

**Необходимые исправления:**
1. Увеличить N до 240 ботов (4×60) для power = 0.80 при d = 0.33
2. Добавить PMID для всех ссылок
3. Устранить seed-зависимость (усреднение по 100 сидам)
4. Заменить ESP32-C3 на ESP32-S3 (больше RAM для TDMA)
5. Включить чел-часы в бюджет

**Сильные стороны:** дешевизна, открытость, контрольные тесты A–F
**Слабые стороны:** нестабильный эффект в симуляции, отсутствие pilot data
